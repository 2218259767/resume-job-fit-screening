# 交付格式

## 文件命名

默认命名：

```text
<site_slug>_resume_match_<YYYYMMDD>.md
```

例如：

```text
meituan_resume_match_20260322.md
```

## 报告结构

按这个顺序写：

1. 标题
   - `# <站点名>岗位匹配报告`
2. 元信息
   - 数据抓取日期
   - 评估口径
   - 搜索范围
   - 用户偏好约束
3. `## 结论先行`
   - 全量岗位数量
   - 按规则直接跳过的数量
   - 候选人真正应该聚焦的岗位主线
4. `## 最推荐的 <topk> 个岗位`
   - 用表格汇总排名、岗位、链接、严格分数、核心推荐理由
5. `## Top <topk> 详细分析`
   - 每个岗位单独一个三级标题
   - 使用严格复筛 prompt 的输出结构
6. `## 其他建议`
   - 投递顺序
   - 简历改写建议
   - 面试准备重点
   - 明确不该高估的岗位类型
7. `## 全量岗位清单`
   - 推荐按 `跳过 / 不建议 / 备选 / 推荐` 分组
   - 每行保留一句理由

## 写作要求

- 先给结论，再给细节。
- 所有结论都要能回溯到 JD 或简历事实。
- 对不完整的数据要明确写出空缺，不要装作已经确认。
- 优先写可投递决策，不要写泛泛而谈的总结。

## 风格参考

如果当前工作区已有历史报告，例如 `meituan_resume_match_20260322.md`，可将其作为结构和颗粒度参考，但不要机械复制原文。
